## 1. 产品摘要

平台服务两类用户：
- **C 端创作者**（博主/营销人员）：用 AI 从 0 快速生成保留个人风格的中文博客，克服"写作启动难、灵感枯竭、AI 味太重"。
- **B 端品牌团队**（内容运营/品牌审核/最终审批）：用 BrandFlow 管理受控知识、品牌规范、渠道规范，以七阶段审批工作流产出多渠道一致的企业内容。

核心价值：从"不知道写什么"到"完成一篇文章"的时间从 3 小时缩短到 30 分钟（C 端）；企业内容从 Brief 到多渠道发布全流程可控、可审计（B 端）。

## 2. 引擎一：AI 写作助手

### 2.1 四步写作流程
1. **选题**：基于关键词/受众生成选题方向
2. **标题生成**：10 个带钩子（数字/问题/对比/悬念）的候选标题，JSON 结构化输出
3. **大纲生成**：按长度（short/medium/long）生成引言 + 主章节 + 结论，含具体子要点
4. **扩写**：Multi-Agent 引擎

### 2.2 Multi-Agent 扩写引擎（核心亮点）
- **Research Agent**：检索 3-5 条近 3 年客观数据/案例，支撑大纲要点
- **Writer Agent**：基于风格卡片（Style Card）撰写，融合数据自然不堆砌
- **Reviewer Agent**：五维度量化评分（大纲覆盖度/数据支撑/逻辑流畅/风格一致/内容合规），不合格输出可执行改写指令，反馈给 Writer 重写，最多 2 轮

### 2.3 风格注入（Few-Shot）
- 用户上传 3-5 篇参考文章 → 提取 Style Card（语气/用词/句式特征）
- 无需 Fine-tune，Prompt 层实现个人风格迁移，解决"AI 味太重"

### 2.4 润色（三级强度）
- light（10% 改动）/ medium（30%）/ deep（60%），参数化控制，负面约束明确

### 2.5 内容合规护栏
- 政治敏感词检测、虚假信息识别、违规即终止（compliance_error）

### 2.6 语义相似度评估
- 用户最终发布版 vs AI 初稿 cosine similarity（text-embedding-3-small）
- 目标区间 0.75-0.85（健康）：<0.5 说明 AI 基本无用，>0.85 需结合满意度区分

## 3. 引擎二：BrandFlow 企业品牌内容工作流

### 3.1 产品原则
- 主界面聚焦内容工作，不聚焦模型配置
- 用户始终知道：阶段、系统动作、人工待办、停止原因、证据
- 审批明确、审慎、绑定版本；关键问题阻塞批准
- 不展示思维链/秘密/完整提示；不使用虚构数据

### 3.2 七阶段任务工作流
```
工作区设置 → 结构化 Brief → 研究与取证 → 大纲审批 → 主内容审阅与审批
→ 多渠道生成与逐一审批 → 最终审批 → MCP 导出/预览
```
- 任一阶段缺失权威事实 → 人工补充（Human-in-the-loop）
- 局部问题 → 块级修订；渠道冲突 → 跨渠道一致性处理

### 3.3 角色与审批矩阵
| 角色 | 主要动作 | 不默认拥有 |
|------|---------|-----------|
| 内容运营 | 建任务/Brief/编辑/提交审批 | 最终审批、授权导出 |
| 品牌审核者 | 语气/术语/渠道一致性审查批准 | 替代作者提交事实、最终风险授权 |
| 最终审批者 | 关键事实/风险/版本比较/批准导出 | 静默改内容后自动批准 |

- 审批绑定不可变版本 + 幂等键；上游变化使受影响决策失效并明示
- 默认禁止提交者/最后编辑者审批自己版本

### 3.4 多渠道内容生成
- 单次任务产出：新闻稿(800-1200字) / 群通知(100-200字) / 员工故事(500-800字) / 活动文案(200-400字)
- 每渠道风格独立，强制展示来源主版本，跨渠道一致性校验

### 3.5 技术支撑
- **FastAPI + LangGraph**：工作流编排、状态机、检查点恢复
- **MCP Server（brand-tools）**：九项品牌工具 + 高风险审批门禁，普通活动与高级脱敏追踪分层呈现
- **评估体系**：坏案例闭环、真实评估数据（禁止虚构指标）

## 8. 安全与合规

- 提示词防泄露（系统指令不暴露）
- 内容合规熔断（政治敏感/虚假信息直接终止）
- 审批鉴权：所有审批/导出服务端按 workspace/租户/角色重鉴权，前端禁用状态不构成安全边界
- 真实数据原则：禁止展示虚构转化/成功率
